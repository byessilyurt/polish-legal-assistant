## Contributing to Documentation

When updating docs:

1. **Keep it current:** Update docs when code changes
2. **Be clear:** Use simple language
3. **Add examples:** Show, don't just tell
4. **Update index:** Add new sections here
5. **Check links:** Ensure all links work
6. **Review style:** Match existing format

### Documentation Style Guide

**Headings:**
- Use # for file title
- Use ## for major sections
- Use ### for subsections
- Use #### for details

**Code Blocks:**
- Use ```bash for commands
- Use ```typescript for code
- Use ```json for data
- Include language identifier

**Lists:**
- Use `-` for unordered lists
- Use `1.` for ordered lists
- Use `[ ]` for checklists

**Links:**
- Use relative links for local files
- Use full URLs for external resources
- Make link text descriptive

**Emphasis:**
- Use **bold** for important terms
- Use *italic* for emphasis
- Use `code` for inline code/commands

## Getting Help

### Can't find what you need?

1. Search all markdown files:
   ```bash
   grep -r "search term" *.md
   ```

2. Check the code comments:
   ```bash
   grep -r "// your question" .
   ```

3. Review Git history:
   ```bash
   git log --all --grep="feature name"
   ```

4. Ask the team:
   - Check team chat
   - Create a GitHub issue
   - Schedule office hours
